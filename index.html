3. Java基礎（Lesson 01）

static と instance

staticはクラスに1つ。インスタンスメンバは各オブジェクトに属する。静的メソッドからインスタンスメンバへ直接アクセスできない。

overload / override

overload: 同名・異なる引数。override: 継承先で同じシグネチャを再定義し、@Overrideを付ける。

abstract / interface

抽象クラスはextends、インタフェースはimplements。抽象メソッドを持つ具象クラスは実装が必要。

クラス構造の基本Lesson 01 / Lesson 14 p.5
class Vector3 extends Vector2 {
  static int count;        // クラスフィールド
  private double z;        // インスタンスフィールド

  Vector3(double x, double y) {
    this(x, y, 0);      // 同じクラスの別コンストラクタ
  }
  Vector3(double x, double y, double z) {
    super(x, y);          // 親コンストラクタ
    this.z = z;
  }
  @Override public String toString() { return "z=" + z; }
}
頻出語: class / instance / constructor / inheritance / superclass / subclass / overload / override / abstract class / interface / recursion。

Swingはjavax.swing。コンポーネントを生成し、コンテナへ追加し、レイアウトを設定し、リスナーを登録する。

public class ButtonFrame extends JFrame
    implements ActionListener {

HelloTest.java - 完成形（写真から再構成）最重要
public class HelloTest extends Application {
  private Label lb;
  private TextField tf;
  private CheckBox ck;

  public void start(Stage stage) throws Exception {
    lb = new Label("Who are you?\nMale or female?");
    lb.setFont(Font.font("Times New Roman", FontWeight.BOLD, 14));
    tf = new TextField("");
    tf.setOnAction(new TextFieldEventHandler());
    ck = new CheckBox("Male");

    HBox hb = new HBox();
    BorderPane bp = new BorderPane();
    hb.getChildren().addAll(tf, ck);
    bp.setTop(lb);
    bp.setCenter(hb);
    Scene scene = new Scene(bp, 300, 130);
    stage.setTitle("Hello");
    stage.setScene(scene);
    stage.show();
  }

  private class TextFieldEventHandler
      implements EventHandler<ActionEvent> {
    public void handle(ActionEvent e) {
      String name = tf.getText();
      if (name.isEmpty()) {
        lb.setText("Please input your name in TextField");
      } else if (ck.isSelected()) {
        lb.setText("Hello, Mr. " + name);
      } else {
        lb.setText("Hello, Ms. " + name);
      }
    }
  }
}
再構成上の注意: 写真ではフォント引数やScene寸法が判読不能なため、それらの数値・書体は等価な例で補っている。得点核はgetChildren().addAll、Pane配置、setScene/show、isSelectedによる分岐。
HelloTestの実行結果を答える
  • 空欄のままEnter → “Please input your name in TextField”
  • Hanako、Male未選択 → “Hello, Ms. Hanako”
  • Ichiro、Male選択 → “Hello, Mr. Ichiro”

7. Thread（Lesson 09）

Threadは軽量なsub-process／処理の最小単位。複数threadは共有メモリ上で独立・並行に処理でき、ネットワーク・ゲーム・アニメーションで使われる。

Runnable方式（最優先）
class Counter implements Runnable {
  public void run() {
    for (int i = 0; i < 10; i++) {
      System.out.println(i);
      try { Thread.sleep(100); }
      catch (InterruptedException e) { return; }
    }
  }
}
Thread t = new Thread(new Counter());
t.start();

必須メソッド

  • start(): 新しい実行経路を開始
  • run(): threadが行う仕事
  • sleep(ms): 指定時間休眠（static）
  • currentThread()、getName()、isAlive()
  • wait()、notify()、notifyAll()

資料にはstop/suspend/resumeも登場するが、現代Javaでは非推奨。試験では資料上の役割を答えられるようにする。

超頻出の区別: t.run()を直接呼ぶだけでは通常のメソッド呼出し。新スレッドとして動かす入口はt.start()。

ライフサイクル

生成 NEW→ start()実行可能 RUNNABLE→実行→待機 / 休眠 / Block→終了

8. Network（Lesson 10-11）- 最重要

体験談画像で名指し: 「Webのクライアントとサーバーが何たらみたいなところのコードも出た」。TCP EchoServer / EchoClient、ServerSocketとSocketの役割、stream生成順を最優先で手書きする。

Client / Server

Serverは要求を待つpassive opener、Clientは接続を開始するactive opener。Socketは通信端点で、概念的にIP address : port number。

TCP / UDP

TCPはstream-basedで接続型。UDPはDatagramSocket、DatagramPacket、InetSocketAddressを使う。

TCP Echo Server

EchoServer.javaLesson 10 p.42-43 / Lesson 11 p.14-15
public class EchoServer {
  public static final int PORT = 9999;
  public static void main(String[] args) throws IOException {
    ServerSocket listenSock = new ServerSocket(PORT);
    try {
      Socket socket = listenSock.accept();
      BufferedReader in = new BufferedReader(
        new InputStreamReader(socket.getInputStream()));
      PrintWriter out = new PrintWriter(
        new BufferedWriter(
          new OutputStreamWriter(socket.getOutputStream())), true);
      String line;
      while ((line = in.readLine()) != null) {
        out.println(line);
        if (line.equalsIgnoreCase("END")) break;
      }
      socket.close();
    } finally { listenSock.close(); }
  }
}

TCP Echo Client

EchoClient.javaLesson 10 p.45-46
InetAddress addr = InetAddress.getByName("localhost");
Socket socket = new Socket(addr, EchoServer.PORT);
BufferedReader in = new BufferedReader(
    new InputStreamReader(socket.getInputStream()));
PrintWriter out = new PrintWriter(
    new BufferedWriter(
      new OutputStreamWriter(socket.getOutputStream())), true);

out.println(line);          // Client → Server
String reply = in.readLine(); // Server → Client
socket.close();
向き: Clientのout ↔ Serverのin、Clientのin ↔ Serverのout。PrintWriter(..., true)のtrueはauto-flush。

複数クライアント

ServerSocket→ accept()通信Socket→ new ThreadClient handler→ start()次のacceptへ
同時処理の骨格
ServerSocket server = new ServerSocket(PORT);
while (true) {
  Socket socket = server.accept();
  Thread worker = new Thread(new ClientHandler(socket));
  worker.start();
}

URL / InetAddress / WebView

URL形式はscheme://host:port/path。InetAddress.getByName、getHostName、getHostAddress、isReachable。JavaFX埋込みブラウザはWebView → getEngine() → webEngine.load(url)。

9. Servlet / JSP（Lesson 12-13）

HTTPの基本

Web client（browser）がHTTP Request、Web serverがHTTP Response。資料ではHTTPをconnection-orientedだがstatelessと説明。Requestはrequest line・headers・optional body、Responseはstatus line・headers・body。

GETPOST
主にURLで資源取得。フォーム値がquery stringに現れやすい。値をrequest bodyへ送り、作成・更新や多量／機微な入力に向く。

Servlet

基本形Lesson 12 / Lesson 14 p.48
public class HelloServlet extends HttpServlet {
  protected void doGet(HttpServletRequest request,
                       HttpServletResponse response)
      throws ServletException, IOException {
    response.setContentType("text/html; charset=UTF-8");
    String name = request.getParameter("name");
    PrintWriter out = response.getWriter();
    out.println("<html><body><h1>Hello " + name + "</h1></body></html>");
    out.close();
  }
}

作成手順: HttpServletを継承 → doGetまたはdoPost → request.getParameter → response.getWriter → HTMLを出力 → writerを閉じる。TomcatはServlet Container。

JSP

記法名称役割
<% code %>scriptletJava文
<%= expression %>expression値を出力
<%! declaration %>declaration変数・メソッド宣言
<%@ ... %>directivepage / include等の全体設定
<jsp:include ... />action実行時に別資源を含める
<jsp:forward ... />action別資源へ処理を転送
JSPフォーム＋session
<%@ page contentType="text/html; charset=UTF-8" %>
<%
  String name = request.getParameter("name");
  session.setAttribute("user", name);
%>
<p>Hello, <%= name %></p>
<p>Session user: <%= session.getAttribute("user") %></p>

暗黙オブジェクト: out、request、response、session、application等。JSPは最終的にServletへ変換・コンパイルされ、サーバー側で実行される。
